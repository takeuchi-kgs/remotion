処理フローの全体像
パイプラインは6つのステップで構成されています。

1. 台本生成
メモからClaude Codeが構造化された台本を生成します。誰が何を話すか、どんな表情で話すか、といった情報が含まれます。

2. スライド定義生成
台本の各シーンには「どんなスライドを表示するか」という情報（slideプロパティ）が含まれています。Node.jsスクリプトがこれを抽出し、スライド定義ファイル（slides.json）にまとめます。タイトル、ポイント解説、ステップ説明など26種類のテンプレートから選択されます。

3. 画像生成
スライドに必要な画像をGemini Imagen APIで生成します。「シンプルなフラットデザインのイラスト」といった指示で、統一感のある画像が作られます。

4. スライドPNG化
Marp CLIを使って、Markdownで書かれたスライドをPNG画像に変換します。

5. 音声生成
VOICEVOXを使って、台本のセリフを音声に変換します。キャラクターごとに異なる声を割り当てられます。

6. 動画レンダリング
Remotion（React製の動画フレームワーク）で全てを組み合わせ、MP4動画として出力します。ブラウザでプレビュー確認後、`remotion render`コマンドでMP4に書き出します。

キャラクターアニメーションの工夫
動画には解説キャラクターが登場します。静止画だと味気ないので、簡易的なアニメーションを入れています。

口パク
4枚の画像を用意しています（目open/close × 口open/close）。音声に合わせて口を開閉させることで、喋っているように見せます。

まばたき
2〜5秒のランダム間隔でまばたきします。これだけで「生きてる感」が出ます。

音声周りの調整
音のバランスも自動で調整しています。

ナレーション: 100%（メイン）

BGM: 15%（控えめに）

効果音: 30〜50%（表現別に調整）

シーン切り替え時には効果音を入れて、メリハリをつけています。

動画制作において、スライドの見た目は視聴者の印象を大きく左右します。これまでMarpを使ってスライドを生成していましたが、いくつかの限界に直面したため、Reactコンポーネントベースの生成方式に移行しました。本記事では、その経緯と具体的な実装方法を解説します。


Marpの限界
Marpは Markdown からスライドを生成できる便利なツールですが、動画制作で使う上では以下の問題がありました。

デザインの自由度が低い - CSSでのカスタマイズに限界がある

アニメーションが使えない - 静的なスライドしか作れない

動画との統一感が出しにくい - 他の要素（キャラクター、字幕）と別系統のデザインになる

細かい調整が大変 - フォントサイズや余白の微調整が困難

特にアニメーションの制限は致命的で、動画内でスライドが「動かない」のは違和感がありました。

新しい方式：Reactコンポーネントによる直接レンダリング
新しい方式では、台本（script.json）からReactコンポーネントで直接スライドを描画します。

台本（script.json） → Reactコンポーネント → 動画内で直接レンダリング
Marpを経由せず、動画フレームワーク（Remotion）の中で直接描画するため、技術スタックが統一されます。

7種類のテンプレート
用途に応じて以下のテンプレートを用意しました。

| テンプレート | 用途 |
|-------------|------|
| TitleSlide | タイトル表示 |
| ListSlide | 箇条書き |
| StepsSlide | 手順・フロー |
| ImageTextSlide | 画像+説明 |
| TableSlide | 比較表 |
| SummarySlide | まとめ |
| EndingSlide | 締め・CTA |

台本に `slide.type` を指定するだけで、適切なテンプレートが自動選択されます。

画像のAI自動生成
スライドに挿入する画像は、Gemini AIで自動生成できます。台本にプロンプトを書くだけで、16:9のスライド用画像が生成されます。

{
  "image": {
    "source": "generate",
    "prompt": "フラットなイラストでメモ帳を描いて"
  }
}
デザイントークンによる統一管理
色、フォント、余白などは「デザイントークン」として一元管理しています。テーマファイルを1箇所修正するだけで、全スライドのデザインが一括で変更できます。

この方式のメリット
動画全体で統一したデザイン - キャラクターや字幕と同じデザインシステム

アニメーションも自由に追加 - Reactなので何でもできる

テンプレート選択で効率化 - 毎回デザインを考える必要がない

新テンプレートの追加も簡単 - Reactコンポーネントを追加するだけ

背景・きっかけ
Remotionで解説動画を自動生成するパイプラインを作っています。台本生成からスライド作成、音声合成まで自動化できたのですが、効果音がないと動画にメリハリが出ませんでした。

「台本に書くだけでSEが自動で入る」仕組みを作れないかと考え、SE機能を実装しました。


全体像
SE機能は3つのステップで動いています。

台本生成（AI） → SE定義ファイル参照 → Remotionで再生

ポイントは「SE定義ファイル」の存在です。AIが台本を生成する際にこのファイルを参照し、シーンに合った効果音を自動で選択します。人間が毎回「ここにこのSEを入れて」と指定する必要がありません。

実際にやったこと
1. 4種類のSEトリガーを設計
SE再生のトリガーは以下の4種類です。

スライドSE: スライドが表示された瞬間に再生されます。セクション切り替え時の「シーン切り替え音」などに使います

セリフトリガー: 特定のセリフが始まるタイミングで再生されます。「やった！」というセリフにレベルアップ音を付けるような使い方です

アノテーショントリガー: スライド内の注釈や比較カラムが表示されるタイミングで再生されます

図表SE: グラフや表が表示されるタイミングで再生されます

基本はスライドSEだけでも十分です。慣れてきたらセリフトリガーや図表SEを組み合わせると、より豊かな演出になります。

2. script.jsonへのSE指定方法
スライドにSEを付ける場合は se プロパティを追加するだけです。

{
  "slide": {
    "type": "bridge",
    "title": "第2章",
    "se": {
      "path": "audio/se/シーン切り替え1.mp3",
      "volume": 0.3
    }
  }
}

copy
セリフにSEを付ける場合も同様です。

{
  "speaker": "left",
  "text": "やった！完成した！",
  "se": {
    "path": "audio/se/レベルアップ.mp3",
    "volume": 0.3
  }
}

copy
volumeは0.3を基本としています。BGMや音声の邪魔にならない程度の音量です。

3. SE定義ファイル（README.md）の作成
public/audio/se/README.md にSE一覧を定義しています。

きらーん1.mp3 - メリット・利点を強調するとき
チーン1.mp3 - シリアスな失敗シーン
データ表示1.mp3 - グラフや表を表示するとき
レベルアップ.mp3 - 目標達成・成功の瞬間
警告音2.mp3 - 重要な警告・注意喚起

copy
このファイルで重要なのは「使用タイミング」の記述です。AIはファイル名だけでは音の印象がわからないため、「どんな場面で使うか」を明記しておくことで選択精度が上がります。

4. Remotionでの再生実装
Remotionの Audio コンポーネントと Sequence を組み合わせて実装しています。5つのSEコンポーネントがそれぞれのトリガータイプに対応しています。

SlideSoundEffect - スライド表示時に即座に再生

LineSoundEffect - セリフ開始時に再生

DiagramSoundEffect - ダイアグラム表示の0.3秒後に再生

AnnotationSoundEffect - セリフトリガーのアノテーションSE

AnnotationTriggerSoundEffect - タイミング自動計算のアノテーションSE

5. 効果音素材の入手
効果音ラボ（soundeffect-lab.info）を使用しています。

無料で商用利用可能です

種類が豊富で、システム音や演出音など解説動画に使いやすい音が揃っています

日本語で検索できるため、欲しい音を見つけやすいです

クレジット表記不要です
